---
title: "search"
description: "Semantic search across all stored facts in the workspace"
---

Runs a semantic search across workspace memories. Returns the most relevant facts with similarity scores. Scope to a specific contact or company, or search workspace-wide.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `q` | string | Yes | Search query in natural language |
| `contact_id` | string | No | Scope to one contact |
| `company_id` | string | No | Scope to one company |
| `limit` | number | No | Max results (default 10, max 20) |

## Returns

```
Search results for "budget" (2):
  [deal] (May 12) [94%] Budget confirmed at $24k ARR
  [deal] (May 8)  [71%] CFO mentioned budget review in June
```

Each result shows: category, when it was stored, similarity score, and the fact content.

## When to call

Use `search` when you have a name or company but not a UUID, or when you want to pull relevant facts before drafting a message. For a known contact, `get_contact` is faster — it returns all their facts in one call. Use `search` for cross-workspace queries or when you're looking for something specific.

<Tip>
The underlying REST endpoint is `POST /v1/search` (POST variant of `GET /v1/memories/search`). Contact-scoped searches use a lower similarity threshold (0.45) to return more results.
</Tip>
