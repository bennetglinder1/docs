---
title: "update_contact"
description: "Update profile fields on an existing contact"
---

Partially updates a contact's profile. Only the fields you provide are changed — everything else is left as-is.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `email` | string | One of | Contact's email (to identify them) |
| `contact_id` | string | One of | Contact UUID |
| `first_name` | string | No | Updated first name |
| `last_name` | string | No | Updated last name |
| `company` | string | No | Updated company name |
| `job_title` | string | No | Updated job title |
| `phone` | string | No | Updated phone number |
| `linkedin_url` | string | No | Updated LinkedIn URL |
| `notes` | string | No | Updated free-form notes |

Either `email` or `contact_id` must be provided to identify the contact.

## Returns

```
Updated contact: Alex Chen
CTO · Acme Corp
contact_id: a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

## When to call

Use when you learn that a contact's role, company, or details have changed. For facts about the relationship (e.g. "they got promoted"), use `remember` instead — it persists the insight as a queryable fact rather than overwriting a field.
