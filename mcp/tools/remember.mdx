---
title: "remember"
description: "Store a fact — about a contact, their company, or your own workspace"
---

Stores a persistent fact in the memory layer. Facts are categorised, deduplicated, and made searchable. Use this for things you want to remember across sessions — not for logging what happened (use `track` for that).

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `fact` | string | Yes | The fact to store — one clear, reusable sentence |
| `email` | string | No | Scope to a contact by email (omit for workspace-level) |
| `contact_id` | string | No | Scope to a contact by UUID |
| `category` | string | No | Memory category (default: `General`) |
| `company` | boolean | No | Set `true` to tag to the whole company, not just this person |

### Memory categories

`ICP` · `Product` · `Pricing` · `Market` · `Competitors` · `Team` · `Patterns` · `General`

## Scoping

- **No email / contact_id** → stored as a workspace-level fact (your ICP, product, pricing, etc.)
- **With email or contact_id** → scoped to that individual
- **With `company: true`** → resolves the contact's `company_id` and scopes to the whole org

## Returns

```
Stored 1 fact in contact memory [deal]: "Budget confirmed at $24k ARR"
```

Similar existing facts in the same scope are automatically superseded — no duplicates accumulate.

## Examples

```
// Contact-level fact
remember({ email: "alex@acmecorp.com", fact: "Concerned about integration complexity", category: "General" })

// Company-level fact (resolves alex's company_id)
remember({ email: "alex@acmecorp.com", fact: "Evaluating 3 vendors, decision by Q2", category: "deal", company: true })

// Workspace-level fact (no contact)
remember({ fact: "Our ICP is B2B SaaS companies with 50-500 employees running outbound sales", category: "ICP" })
```

<Tip>
The underlying REST endpoint is `POST /v1/remember`. Facts → `remember`. Events → `track`.
</Tip>
