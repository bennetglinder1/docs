---
title: "track"
description: "Record that something happened — a call, email, meeting, website visit"
---

Logs an activity event against a contact. The pipeline stage may advance automatically based on the activity type — a DB trigger evaluates the signal and promotes the stage if thresholds are met.

Use `track` for **what happened**. For **what you learned**, also call `remember`.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `email` | string | One of | Contact's email address |
| `contact_id` | string | One of | Contact UUID |
| `type` | string | Yes | Activity type (see below) |
| `description` | string | No | Brief summary of what happened |
| `occurred_at` | string | No | ISO timestamp — defaults to now |

### Activity types

| Type | When to use |
|------|-------------|
| `email_sent` | You sent an outbound email |
| `email_reply` | They replied to your email |
| `call_held` | A call took place |
| `meeting_held` | An in-person or video meeting |
| `linkedin_message` | Sent or received a LinkedIn message |
| `linkedin_connected` | Connection request accepted |
| `follow_up_sent` | Sent a follow-up (non-initial) |
| `proposal_sent` | Sent a proposal or quote |
| `website_visit` | They visited your website (from RB2B etc.) |
| `content_download` | Downloaded a resource |
| `trial_started` | Started a free trial |
| `manual_note` | Anything else worth logging |

## Returns

```
Logged `meeting_held`: "30-min discovery call, covered pricing and timeline".
```

## When to call

Call `track` immediately after any meaningful touchpoint. It keeps the activity timeline accurate, feeds the pipeline stage logic, and ensures `get_contact` returns an up-to-date picture.

<Tip>
The underlying REST endpoint is `POST /v1/capture` (also aliased as `POST /v1/track`). The MCP tool exposes a curated set of activity types — the REST endpoint accepts any string as `type`.
</Tip>
