---
title: "track"
description: "Record that something happened ‚Äî a call, email, meeting, website visit"
---

Logs an activity event against a contact. The pipeline stage may advance automatically based on the activity type ‚Äî a DB trigger evaluates the signal and promotes the stage if thresholds are met.

Use `track` for **what happened**. For **what you learned**, also call `remember`.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `email` | string | One of | Contact's email address |
| `contact_id` | string | One of | Contact UUID |
| `type` | string | Yes | Activity type (see below) |
| `description` | string | No | Brief summary of what happened |
| `occurred_at` | string | No | ISO timestamp ‚Äî defaults to now |

### Activity types

| Type | When to use |
|------|-------------|
| `email_sent` | You sent an outbound email |
| `email_reply` | They replied to your email |
| `call_held` | A call took place |
| `meeting_held` | An in-person or video meeting |
| `linkedin_message` | Sent or received a LinkedIn message |
| `linkedin_connected` | Connection request accepted |
| `follow_up_sent` | Sent a follow-up (non-initial) |
| `proposal_sent` | Sent a proposal or quote |
| `website_visit` | They visited your website (from RB2B etc.) |
| `content_download` | Downloaded a resource |
| `trial_started` | Started a free trial |
| `manual_note` | Anything else worth logging |

## Returns

The response always confirms what was logged. If the DB trigger advanced the pipeline stage, that's reported too. A contextual next-step hint is appended based on the activity type:

```
Logged `call_held`: "30-min discovery call, covered pricing and timeline".
Stage advanced: interested ‚Üí evaluating.
Next: Call remember if you learned something worth keeping.
```

```
Logged `proposal_sent`: "Q2 outbound package ‚Äî ‚Ç¨1,200/mo".
Next: Flag for follow-up if no reply in 3 days.
```

### Next-step hints by activity type

| Activity | Hint |
|----------|------|
| `call_held`, `meeting_held` | Call `remember` if you learned something worth keeping |
| `email_reply`, `linkedin_replied` | They replied ‚Äî draft your response or update the pipeline stage |
| `proposal_sent` | Flag for follow-up if no reply in 3 days |
| `follow_up_sent` | Consider updating the pipeline stage if they're not engaging |
| `trial_started` | High-intent signal ‚Äî consider moving to evaluating stage |
| `linkedin_connected` | Good opener ‚Äî send an intro message |

## When to call

Call `track` immediately after any meaningful touchpoint. It keeps the activity timeline accurate, feeds the pipeline stage logic, and ensures `get_contact` returns an up-to-date picture.

<Tip>
The underlying REST endpoint is `POST /v1/capture` (also aliased as `POST /v1/track`). The MCP tool exposes a curated set of activity types ‚Äî the REST endpoint accepts any string as `type`.
</Tip>
