---
title: "delete_memory"
description: "Remove an outdated or incorrect memory"
---

Soft-deletes a memory by ID — marks it inactive so it no longer appears in `get_memories` or `search`. Use to clean up facts that are wrong, superseded, or no longer relevant.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `memory_id` | string | Yes | Memory UUID — get this from `get_memories` output |

## Returns

```
Deleted memory (m1b2c3d4-...): "Budget confirmed at $18k ARR"
```

## When to call

When `get_memories` or `search` surfaces a fact that's outdated or incorrect. Get the `memory_id` from the output, then delete it and store the corrected fact with `remember`.

<Warning>
Deletion is soft — the record is marked `is_active: false` in the database. It can be recovered directly from Supabase if needed.
</Warning>
