---
title: "list_contacts"
description: "Find contacts by pipeline stage — useful for identifying who to prioritise or reach out to next"
---

Returns a list of contacts, optionally filtered by pipeline stage. Use this when you need to work across a segment — e.g. "everyone in `evaluating`" or "all clients for a check-in".

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `stage` | string | No | Pipeline stage to filter by. Omit to list all. |
| `limit` | number | No | Max contacts to return (default 20, max 50) |

### Stage values

`identified` → `aware` → `interested` → `evaluating` → `client`

## Returns

Formatted list of contacts:

```
Contacts — evaluating (4 of 4):
  Alex Chen @ Acme Corp — evaluating  ICP:87  (a1b2c3d4-...)
  Sarah Kim @ Globex — evaluating  ICP:74  (c3d4e5f6-...)
  James Park @ Initech — evaluating  ICP:68  (e5f6a7b8-...)
  Priya Sharma @ Umbrella — evaluating  ICP:61  (f6a7b8c9-...)
```

Each line includes: name, company, stage, ICP score, and `contact_id`.

## When to call

Use `list_contacts` to identify a group to work through. Then call `get_contact` on each individual before acting on them.

<Tip>
The underlying REST endpoint is `GET /v1/contacts`. Additional filters (search query, company_id, LinkedIn URL) are available via the REST API but not exposed in this MCP tool.
</Tip>
