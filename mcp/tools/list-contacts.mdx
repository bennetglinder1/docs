---
title: "list_contacts"
description: "Find contacts sorted by urgency — high ICP + longest since last touch first"
---

Returns contacts sorted by urgency: high ICP score penalised by days since last touch. High-ICP contacts that have gone cold bubble to the top. Use this when you need to decide who to work on next, or to find who's slipping through the cracks.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `stage` | string | No | Filter by pipeline stage. Omit to list all. |
| `filter` | string | No | `hot` = active in last 14d with health ≥ 45; `engaged` = active in last 60d |
| `limit` | number | No | Max contacts to return (default 20, max 50) |

### Stage values

`identified` → `aware` → `interested` → `evaluating` → `client`

## Returns

Contacts sorted by urgency, with last-touch age and a `!!` flag for stale high-ICP contacts:

```
Contacts — evaluating (8 of 23, sorted by urgency):
!! = high ICP, gone cold
!! Vusal Ibrahimli @ Left Click AI — evaluating · ICP:82 · 9d ago (a1b2c3d4-...)
!! James Okafor @ Stripe — evaluating · ICP:77 · 14d ago (c3d4e5f6-...)
   Kemal Heydarov @ TechCorp — evaluating · ICP:71 · 2d ago (e5f6a7b8-...)
   Sara Liu @ Notion — evaluating · ICP:55 · today (f6a7b8c9-...)
```

Each line includes: name, company, stage, ICP score, last-touch age, and `contact_id`. Contacts marked `!!` have an ICP score ≥ 70 and haven't been touched in over 7 days.

## Urgency sort

The tool fetches up to 50 contacts and re-sorts them before returning your requested `limit`. Sort formula:

```
urgency = icp_score - min(days_since_last_touch × 1.5, 50)
```

A contact with ICP 80 last touched 20 days ago (urgency 50) ranks above one with ICP 60 touched today (urgency 60) — recency matters, but not more than fit.

## When to call

Use `list_contacts` to identify who to prioritise. Contacts marked `!!` are your highest-leverage outreach opportunities — strong fit, gone cold. Then call `get_contact` on each individual before acting on them.

<Tip>
The underlying REST endpoint is `GET /v1/contacts`. Additional filters (search query, company_id, LinkedIn URL, sort) are available via the REST API but not exposed in this MCP tool.
</Tip>
