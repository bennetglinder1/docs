---
title: "get_memories"
description: "Load workspace-level knowledge — your ICP, product, pricing, and competitive intel"
---

Returns all workspace-level facts grouped by category. These are the things your agents need to know about *your* company before they can act intelligently — your ICP, what your product does, how you're priced, and who you compete against.

This is not for contact or company facts — use `get_contact` or `get_company` for those.

## Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `category` | string | No | Filter to one category — omit to get all |
| `limit` | number | No | Max facts to return (default 50, max 200) |

### Category values

`ICP` · `Product` · `Pricing` · `Market` · `Competitors` · `Team` · `Patterns` · `General`

## Returns

Facts grouped by category:

```
Workspace knowledge (6 facts):

[ICP]
  • B2B SaaS companies, 50–500 employees, running outbound sales
  • Typical buyer: Head of Sales or RevOps, not the CEO

[Product]
  • Proply is a CRM built for AI agents — it's the memory layer that agents read and write

[Pricing]
  • $99/mo for Growth plan, $249/mo for Scale — no per-seat fees

[Competitors]
  • HubSpot: enterprise, expensive, not agent-native
  • Attio: design-forward, no AI memory layer
```

## When to call

Call `get_memories` at the start of any outreach or pitch task. It gives the agent the product and market context it needs to write relevant, accurate messages — without you having to re-explain your company in every prompt.

<Tip>
Store workspace facts using `remember` without a contact — e.g. `remember({ fact: "Our ICP is...", category: "ICP" })`.
</Tip>
