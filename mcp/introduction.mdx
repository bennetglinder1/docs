---
title: "MCP Server"
description: "Connect any MCP-compatible AI agent to Proply's contact intelligence layer"
---

The Proply MCP server exposes your CRM as a set of tools that AI agents (Claude, GPT-4o, Cursor, etc.) can call directly. Agents get structured contact data, activity timelines, and a persistent memory layer — without any prompt engineering.

## Install

```bash
npm install -g @goproply/mcp
```

Or run without installing:

```bash
npx @goproply/mcp
```

## Configuration

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `PROPLY_API_KEY` | Yes | — | Your workspace API key (Settings → API Keys) |
| `PROPLY_API_URL` | No | `https://api.goproply.com` | Override for self-hosted instances |

## Connect to Claude Desktop

Add to `claude_desktop_config.json`:

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "proply": {
      "command": "npx",
      "args": ["@goproply/mcp"],
      "env": {
        "PROPLY_API_KEY": "pk_live_xxxxxxxx"
      }
    }
  }
}
```

## Connect to Cursor

Add to `.cursor/mcp.json` in your project root (or `~/.cursor/mcp.json` globally):

```json
{
  "mcpServers": {
    "proply": {
      "command": "npx",
      "args": ["@goproply/mcp"],
      "env": {
        "PROPLY_API_KEY": "pk_live_xxxxxxxx"
      }
    }
  }
}
```

## Self-hosted

Point the server at your own instance via `PROPLY_API_URL`:

```json
{
  "env": {
    "PROPLY_API_KEY": "pk_live_xxxxxxxx",
    "PROPLY_API_URL": "https://api.yourdomain.com"
  }
}
```

## Available tools

| Tool | What it does |
|------|-------------|
| `get_contact` | Full contact profile — status line, direction arrows, tiered activity timeline, facts with age |
| `list_contacts` | Contacts sorted by urgency (high ICP + gone cold first), with `!!` flag for stale high-fit contacts |
| `create_contact` | Add a new contact |
| `update_contact` | Update profile fields |
| `delete_contact` | Remove a contact permanently |
| `get_company` | Company profile + all contacts at the account |
| `track` | Log an interaction — returns stage advancement + a contextual next-step hint |
| `remember` | Store a fact about a contact or company |
| `get_memories` | Load workspace-level knowledge (ICP, product, pricing) |
| `search` | Semantic search across all stored facts |
| `delete_memory` | Remove an outdated or incorrect memory |

## Recommended agent workflow

```
Finding who to work on:
  → list_contacts  (sorted by urgency — start with !! contacts)

Before acting on a contact:
  → get_contact    (status, timeline, facts in one call)
  → get_memories   (your ICP + product context before drafting)

After every interaction:
  → track    (what happened — follow the next-step hint it returns)
  → remember (what you learned)
```
