## The problem Proply solves

Your GTM stack has a data problem. Apollo has the prospect list. Salesforce has the deal history. Smartlead has the email sequences. Gmail has the replies. LinkedIn has the conversations. None of these systems know what's happening in the others — and none of them are queryable by an AI agent in a structured way.

The result: duplicate contacts, missed signals, agents that act without context, and attribution that's always wrong.

## How Proply works

Proply sits underneath your GTM stack as the identity layer. It ingests signals from every connected tool, resolves them to a single record per human, and exposes the full account context to agents through MCP.

**Three steps:**

1. **Ingest** — Connect your GTM tools. Proply pulls signals from Apollo, Salesforce, Smartlead, Gmail, LinkedIn, and any custom webhook in real time.
2. **Resolve** — Every signal is matched to a single contact record via email, LinkedIn URL, phone, and name+company matching. No duplicates.
3. **Expose** — The unified record is queryable by agents via MCP in a single call — or via REST API from any system.

## Core components

### Signal ingestion

Proply connects to your GTM tools via OAuth, native integrations, and HMAC-authenticated webhooks. Every event — email sent, LinkedIn message, CRM update, sequence enrolled — flows into Proply in real time.

Supported sources:

| Category | Tools |
|---|---|
| CRM | Salesforce, HubSpot, Pipedrive, Attio, Folk, Close |
| Outbound | Smartlead, Instantly, Apollo sequences |
| Email | Gmail, Outlook |
| Social | LinkedIn (via Unipile) |
| Enrichment | Apollo, Prospeo |
| Public signals | RB2B (website visits), Signalbase (intent/funding/job changes) |
| Custom | Any HTTP webhook with HMAC auth |

### Identity resolution

Every inbound signal is resolved to a single contact record. Proply deduplicates across email address, LinkedIn URL, phone number, and name+company matching — so the same person showing up in Apollo, Salesforce, and Gmail becomes one clean record, not three.

### Unified record

Each resolved contact has:

- **Person graph** — Full activity timeline: every email, call, LinkedIn message, sequence event, and CRM update in chronological order
- **Contact memory** — Structured facts synthesized from signals: objections raised, topics discussed, preferences noted
- **Company memory** — Org-level context shared across all stakeholders at an account
- **ICP score** — How well this contact matches your ideal customer profile, recalculated as new signals arrive

### Live sync

Changes in Proply propagate back to connected tools. Update a contact's stage in Proply → it updates in Salesforce. Enrich a contact → Apollo and your CRM stay current. Your stack stays in sync without manual work.

### MCP connector

The MCP connector exposes the full unified record to AI agents. One call returns everything: contact facts, company context, full activity timeline, ICP score, deal stage.

```json
{
  "tool": "get_contact",
  "contact": "sarah@acmecorp.com",
  "response": {
    "name": "Sarah Chen",
    "company": "Acme Corp",
    "stage": "evaluating",
    "icp_score": 87,
    "last_touchpoint": "LinkedIn message — 3 days ago, no reply",
    "memory": "Raised pricing concern on first call. Decision by end of Q2.",
    "company_memory": "3 active sequences running to this account. VP of Sales is the real buyer."
  }
}
```

## Features

| Feature | What it does |
|---|---|
| **Person graph** | Full context and activity history per contact — every touchpoint across all channels |
| **MCP connector** | One call returns complete account context to any MCP-compatible agent |
| **ICP scoring** | Automatically scores contacts against your ideal customer profile as signals arrive |
| **Inbound enrichment** | Cleans and enriches inbound leads before they hit your CRM |
| **Pattern analysis** | Reports on wins, losses, and what campaigns are actually driving pipeline |
| **CRM sync** | Stays fully in sync with Salesforce, HubSpot, Pipedrive — enriches them, doesn't replace them |
