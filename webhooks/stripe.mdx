---
title: Stripe Webhook
description: Log payments from Stripe and automatically promote contacts to client stage.
---

## Endpoint

```
POST https://api.yourdomain.com/inbound/stripe/:workspaceId
```

## Auth

Set `STRIPE_INBOUND_WEBHOOK_SECRET` in `proply.env`. Proply checks `x-proply-signature`.

## Setup in Stripe

1. In Stripe, go to **Developers → Webhooks → Add endpoint**.
2. Set the endpoint URL to your Proply webhook URL.
3. Select these events:
   - `payment_intent.succeeded`
   - `invoice.paid`
   - `charge.succeeded`
4. Copy the signing secret and set it as `STRIPE_INBOUND_WEBHOOK_SECRET` in `proply.env`.

## Supported events

| Stripe event | Activity logged |
|---|---|
| `payment_intent.succeeded` | `payment_received` |
| `invoice.paid` | `payment_received` |
| `charge.succeeded` | `payment_received` |

## How contact matching works

Proply extracts the customer email from (in priority order):
1. `receipt_email`
2. `customer_email`
3. `customer_details.email`
4. `billing_details.email`

The contact must already exist — Stripe events do **not** create new contacts.

## Activity logged

```json
{
  "type": "payment_received",
  "source": "stripe",
  "description": "Payment received: USD 299",
  "raw_data": {
    "event_type": "invoice.paid",
    "amount": 299,
    "currency": "USD",
    "stripe_id": "in_abc123"
  }
}
```

<Note>
  A `payment_received` activity automatically promotes the contact to **Client** stage if they aren't already there.
</Note>
