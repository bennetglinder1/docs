## How webhooks work

Proply's worker exposes a webhook server at `https://api.yourdomain.com/inbound/`. Each source gets its own path with your workspace ID embedded in the URL.

```
https://api.yourdomain.com/inbound/<source>/<workspaceId>
```

All handlers return `{ "ok": true }` immediately — retries won't cause duplicate activities because every event is deduplicated by `external_id`.

## HMAC verification

Set a shared secret for each source in `proply.env`. Proply validates the `x-proply-signature` or `x-hub-signature-256` header using `HMAC-SHA256`.

```bash
# proply.env
LINKEDIN_WEBHOOK_SECRET=your-secret
INSTANTLY_WEBHOOK_SECRET=your-secret
CALENDLY_WEBHOOK_SECRET=your-secret
FIREFLIES_WEBHOOK_SECRET=your-secret
RB2B_WEBHOOK_SECRET=your-secret
STRIPE_INBOUND_WEBHOOK_SECRET=your-secret
```

If a secret is **not set**, the signature check is skipped — fine for local dev, but set secrets in production.

## Finding your workspace ID

Your workspace ID is a UUID visible in **Settings → Workspace**. It's also returned in all API responses under `workspace_id`.
