## Authentication

Every request requires a workspace API key. Generate one in **Settings → API Keys**.

Pass it in either header:

```http
X-API-Key: pk_live_xxxxxxxx
```

```http
Authorization: Bearer pk_live_xxxxxxxx
```

## Base URL

```
https://api.goproply.com
```

All endpoints are prefixed with `/v1`. Self-hosted users can point to their own instance by replacing this with their API domain.

## Errors

All errors return JSON with an `error` field.

| Status | Code | Meaning |
|--------|------|---------|
| `400` | `email_required` / `content_required` | Missing required field |
| `401` | `api_key_required` / `invalid_api_key` | Auth failed |
| `404` | `contact_not_found` / `company_not_found` | Resource not found |
| `409` | `email_already_exists` | Duplicate on create |
| `500` | `internal_error` | Server error |

## Quick examples

<CodeGroup>
```bash List contacts
curl https://api.goproply.com/v1/contacts \
  -H "X-API-Key: pk_live_xxxxxxxx"
```

```bash Get context for an agent prompt
curl "https://api.goproply.com/v1/contacts/alex@acmecorp.com/context" \
  -H "X-API-Key: pk_live_xxxxxxxx"
```

```bash Log a signal
curl -X POST https://api.goproply.com/v1/capture \
  -H "X-API-Key: pk_live_xxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"type":"proposal_viewed","email":"alex@acmecorp.com","description":"Opened proposal 3x"}'
```

```bash Save a memory
curl -X POST https://api.goproply.com/v1/remember \
  -H "X-API-Key: pk_live_xxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"text":"Budget confirmed at $24k ARR","category":"deal","email":"alex@acmecorp.com"}'
```
</CodeGroup>

## Identifier flexibility

The `GET /v1/contacts/:id` endpoint (and several others) accepts either a UUID **or an email address**. AI agents that don't have the UUID can pass the email directly:

```
GET /v1/contacts/alex@acmecorp.com/context
```

## Pipeline stages

Contact pipeline stages are: `identified` → `aware` → `interested` → `evaluating` → `client`

Stages may advance automatically when you log activities via `POST /v1/capture` — a database trigger evaluates the signal and promotes the stage if thresholds are met.
