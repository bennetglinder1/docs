## Overview

Fireflies works in two ways inside Proply:

1. **API key** — Proply can search and retrieve transcripts from Fireflies on demand (e.g. when a contact is looked up by the agent)
2. **Webhook** — Fireflies pushes a notification when a meeting recording is ready; Proply logs a `meeting_held` activity on matched contacts

Both are independent — you can set up one or both.

## API key setup

### 1. Get your Fireflies API key

In Fireflies, go to **Integrations → API** and copy your API key.

### 2. Connect in Proply

Go to **Settings → Integrations → Fireflies**, paste your API key, and click **Connect**. Proply verifies it by calling the Fireflies GraphQL API.

Once connected, the Proply agent can search your meeting transcripts when looking up a contact.

## Webhook setup

The webhook logs meetings automatically as they happen — no polling needed.

### 1. Get your webhook URL

```
https://api.yourdomain.com/inbound/fireflies/<YOUR_WORKSPACE_ID>
```

### 2. Add in Fireflies

Go to **Integrations → Webhooks** in Fireflies and add the URL above.

### 3. Set the HMAC secret

```bash
FIREFLIES_WEBHOOK_SECRET=your-signing-secret
```

Restart the worker after updating.

## What gets logged

`meeting_held` on every matched participant contact.

**Update-only** — Fireflies never creates new contacts from participants.
