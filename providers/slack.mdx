## Overview

Connect Slack via OAuth and Proply polls your direct messages every hour, matching DMs to existing contacts and logging them as `slack_message` activities on their timeline.

**Update-only** — Slack never creates new contacts. DMs are only logged if the person's email matches an existing contact in your workspace.

## Setup

### 1. Register a Slack app (self-hosted only)

<Note>
  On the cloud version, Slack is pre-configured — skip to step 2.
</Note>

1. Go to [api.slack.com/apps](https://api.slack.com/apps) and create a new app.
2. Under **OAuth & Permissions**, add these **User Token Scopes** (not Bot Scopes):
   - `channels:history`
   - `im:history`
   - `im:read`
   - `users:read`
   - `users:read.email`
3. Add your redirect URI: `https://api.yourdomain.com/api/oauth/slack/callback`
4. Copy the **Client ID** and **Client Secret** into `proply.env`:

```bash
SLACK_CLIENT_ID=your-client-id
SLACK_CLIENT_SECRET=your-client-secret
SLACK_REDIRECT_URI=https://api.yourdomain.com/api/oauth/slack/callback
```

### 2. Connect in Proply

Go to **Settings → Integrations → Slack** and click **Connect**. You'll be redirected to Slack to authorize the app.

<Warning>
  Proply needs a **user token** (xoxp-), not a bot token. Make sure to authorize as yourself, not as a bot.
</Warning>

### 3. Done

Proply starts polling your DMs on the next hourly cycle.

## How it works

Every hour, Proply:

1. Lists all your direct message conversations
2. Fetches messages from the last 2 hours
3. Resolves the DM partner's email against your contacts
4. Logs a `slack_message` activity for each matched DM

## Environment variables

| Variable | Description |
|---|---|
| `SLACK_CLIENT_ID` | OAuth app client ID |
| `SLACK_CLIENT_SECRET` | OAuth app client secret |
| `SLACK_REDIRECT_URI` | Must be `https://api.yourdomain.com/api/oauth/slack/callback` |
