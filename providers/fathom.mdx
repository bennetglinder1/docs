## Overview

Fathom sends a webhook when a meeting recording is ready. Proply receives it, matches the calendar invitees to existing contacts, and logs a `meeting_held` activity on each matched contact's timeline.

**Update-only** — Fathom never creates new contacts.

## Setup

### 1. Get your webhook URL

Your Fathom webhook endpoint is:
```
https://api.yourdomain.com/inbound/fathom/<YOUR_WORKSPACE_ID>
```

Find your workspace ID in **Settings → Workspace**.

### 2. Add the webhook in Fathom

1. In Fathom, go to **Settings → Integrations → Webhooks**.
2. Click **Add endpoint** and paste your URL.
3. Select the **Recording ready** event.
4. Copy the signing secret (starts with `whsec_`).

### 3. Set the signing secret

Add it to `proply.env`:

```bash
FATHOM_WEBHOOK_SECRET=whsec_your-signing-secret
```

Restart the worker after updating.

## What gets logged

For each calendar invitee whose email matches a contact in your workspace:

| Field | Value |
|---|---|
| Activity type | `meeting_held` |
| Title | Meeting title from Fathom |
| Summary | Fathom AI summary (first 500 chars) |
| Source | `fathom` |
