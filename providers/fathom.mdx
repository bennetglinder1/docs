## Overview

Fathom works in two ways inside Proply:

1. **API key** — Proply can search and retrieve meeting summaries from Fathom on demand
2. **Webhook** — Fathom pushes a notification when a recording is ready; Proply logs a `meeting_held` activity on matched contacts

Both are independent — you can set up one or both.

## API key setup

### 1. Get your Fathom API key

In Fathom, go to **Settings → Integrations → API** and generate an API key.

### 2. Connect in Proply

Go to **Settings → Integrations → Fathom**, paste your API key, and click **Connect**. Proply verifies it against the Fathom meetings endpoint.

Once connected, the Proply agent can search your Fathom recordings when looking up a contact.

## Webhook setup

The webhook logs meetings automatically as they complete.

### 1. Get your webhook URL

```
https://api.yourdomain.com/inbound/fathom/<YOUR_WORKSPACE_ID>
```

### 2. Add in Fathom

Go to **Settings → Integrations → Webhooks** in Fathom, add the URL above, and select the **Recording ready** event. Copy the signing secret (starts with `whsec_`).

### 3. Set the signing secret

```bash
FATHOM_WEBHOOK_SECRET=whsec_your-signing-secret
```

Restart the worker after updating.

## What gets logged

`meeting_held` on every matched calendar invitee contact.

**Update-only** — Fathom never creates new contacts from meeting participants.
