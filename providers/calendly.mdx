## Overview

Calendly works in two ways inside Proply:

1. **Personal access token** — Proply can query your Calendly data (event types, scheduled events) on demand
2. **Webhook** — Calendly pushes booking and cancellation events; Proply logs them as `meeting_scheduled` / `meeting_cancelled` activities and **creates new contacts** from bookings

Both are independent — you can set up one or both.

## Personal access token setup

### 1. Generate a token

In Calendly, go to **Integrations → API & Webhooks → Personal Access Tokens** and generate a new token.

### 2. Connect in Proply

Go to **Settings → Integrations → Calendly**, paste your token, and click **Connect**. Proply verifies it against `/users/me`.

## Webhook setup

The webhook is the core signal — every new booking creates or updates a contact and logs the meeting.

### 1. Get your webhook URL

```
https://api.yourdomain.com/inbound/calendly/<YOUR_WORKSPACE_ID>
```

### 2. Add in Calendly

Go to **Integrations → API & Webhooks → Webhooks** (requires Professional plan or higher), click **Create Webhook**, and add your URL.

Select events:
- `invitee.created` — new booking
- `invitee.canceled` — cancellation

### 3. Set the HMAC secret

```bash
CALENDLY_WEBHOOK_SECRET=your-signing-secret
```

Restart the worker after updating.

## What gets logged

| Event | Activity | Creates contact? |
|---|---|---|
| `invitee.created` | `meeting_scheduled` | **Yes** — a booking is strong intent |
| `invitee.canceled` | `meeting_cancelled` | No |
