## Overview

Proply receives LinkedIn events via [Unipile](https://unipile.com) — a LinkedIn API provider. You connect your LinkedIn account through Unipile, then manually register two webhooks in the Unipile dashboard pointing at your Proply instance.

Messages and new connections are logged as `linkedin_message` and `linkedin_connection` activities on matching contacts. LinkedIn **creates new contacts** when a message or connection comes from someone not yet in your workspace.

## Prerequisites

- A Unipile account with a connected LinkedIn account
- `UNIPILE_API_KEY` and `UNIPILE_DSN` set in `proply.env`

## Setup

### Step 1 — Connect LinkedIn in Proply

Go to **Settings → Integrations → LinkedIn** and click **Connect**. You'll be redirected to Unipile's hosted OAuth flow to authorize your LinkedIn account.

### Step 2 — Get your webhook URL

Your LinkedIn webhook endpoint is:

```
https://api.yourdomain.com/inbound/linkedin?workspace_id=YOUR_WORKSPACE_ID&secret=YOUR_LINKEDIN_WEBHOOK_SECRET
```

- **`workspace_id`** — find this in **Settings → Workspace**
- **`secret`** — must match `LINKEDIN_WEBHOOK_SECRET` in your `proply.env`

Set the secret first:
```bash
# proply.env
LINKEDIN_WEBHOOK_SECRET=any-random-string-you-choose
```

### Step 3 — Register two webhooks in Unipile

In your Unipile dashboard, go to **Webhooks** and create **two separate webhooks**, both pointing to the same URL above:

**Webhook 1 — Connections**
- Scope: `connections` / All accounts
- Event: **On new relation**
- URL: your Proply webhook URL

**Webhook 2 — Messages**
- Scope: `messages` / All accounts
- Events: **On new message** (select all message events you want)
- URL: your Proply webhook URL

<Note>
  Both webhooks use the same URL — Proply reads the event type from the payload and routes accordingly.
</Note>

### Step 4 — Set env vars and restart

```bash
# proply.env
UNIPILE_API_KEY=your-unipile-api-key
UNIPILE_DSN=your-unipile-dsn-host
LINKEDIN_WEBHOOK_SECRET=the-same-secret-from-step-2
```

Restart the worker after updating.

## Events logged

| Unipile event | Activity logged | Creates contact? |
|---|---|---|
| `new_relation` | `linkedin_connection` | Yes |
| `message_received` | `linkedin_message` | Yes |

## Environment variables

| Variable | Description |
|---|---|
| `UNIPILE_API_KEY` | Your Unipile API key |
| `UNIPILE_DSN` | Your Unipile DSN hostname (e.g. `api3.unipile.com:13005`) |
| `LINKEDIN_WEBHOOK_SECRET` | Shared secret embedded in your webhook URL |
